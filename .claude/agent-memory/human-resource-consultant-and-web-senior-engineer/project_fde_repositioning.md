저장소는 단일 파일(`index.html`)로 구성된 정적 웹 이력서. 별도 프레임워크/빌드 없이 순수 HTML/CSS/JS.

**진행 상황**
- 커밋 `c446014` ("feat: reposition resume for FDE and internal tools roles", 2026-07-05)에서 1차 FDE 리포지셔닝 완료: 타이틀을 "Backend Engineer · FDE / Internal Tools Focus"로 변경, "FDE 방식으로 일하는 이유" 섹션과 "대표 문제 해결 사례"(Emergency Backoffice, Morning Report Alpha) 섹션 추가, 톤을 문제→판단→실행→결과 서사로 재구성.
- 2026-07-05 대화에서 2차로 상세 분석 진행: 현재 이력서와 일반적 FDE 요구 역량(고객 대면, 풀스택/데이터 엔지니어링, 빠른 프로토타이핑, 커뮤니케이션) 간 갭 분석 수행. 핵심 갭: client-facing 경험 부재(사내 운영팀 대상 경험뿐), 프론트엔드/시각화 스킬 명시 부족, 영어/글로벌 커뮤니케이션 언급 없음, FDE 근거 사례가 1개(Emergency Backoffice)뿐이라 다소 얇음.

**Why**: 사용자가 FDE 포지션 지원을 준비 중이며, 실제로 그 역할을 해본 적은 없어 정직하게 강점(문제 정의력, 신속한 실행, 도메인 다양성)을 근거로 포지셔닝해야 함. 없는 경험을 지어내지 않는 것이 중요.

**How to apply**: 향후 이 저장소에서 이력서 수정 작업을 할 때는 [[feedback-analysis-before-edit]]에 따라 큰 변경 전 분석/합의를 먼저 거칠 것. 실제 파일 수정 시 위 갭들(고객 커뮤니케이션 사례 보강, 프론트/시각화 스택 명시, 영어 역량 언급 여부 확인, 정량 지표 보강)을 반영 대상으로 우선 검토.
