이력서/경력기술서 관련 작업을 요청할 때, 사용자는 "우선 분석을 진행하고 결과를 알려달라(대대적인 수정을 바로 하지 말고, 분석부터)"는 방식을 명시적으로 선호했다.

**Why**: 이력서는 개인 브랜딩이 걸린 민감한 문서라, 방향성에 대한 합의 없이 파일을 먼저 고치면 되돌리기 부담이 크고 취향이 갈릴 수 있음. 분석 단계에서 HR 컨설턴트 관점(포지셔닝/갭 분석)과 시니어 웹 엔지니어 관점(HTML/CSS/접근성/트렌디함)을 모두 포함해달라는 요청도 있었음.

**How to apply**: 이력서 관련 요청이 들어오면, 명시적으로 "지금 바로 수정해달라"는 지시가 없는 한 먼저 분석 리포트(현재 상태 요약 → 갭 분석 → 구체적 제안 → 웹 이력서 기술적 개선점)를 제공하고, 실제 Edit/Write는 사용자의 확인 후에 진행할 것.
